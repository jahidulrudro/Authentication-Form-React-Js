"Formalin" is an authentication flow created using React.js and care. It is fully responsive template based on Twitter Bootstrap framework. It also uses the extensive Font Awesome Icons for its various icons. Its not only responsive whereas it is retina ready also, now no blurry images on your HiDPI and retina devices.

How to use Formalin?

"Formalin" is very easy to update and modify, it will adapt your website and brand in no time. Steps for modification and updating are described step by step in this exclusive documentation.

Files and folders structure

Whole app is divided in to several directories and subdirectories. The whole app consists of React components, subcomponents. The whole directory structure is given below. The work of every component and subcomponent can be easily understood by their name.

P.S. This directory structure is adopted from create-react-app. 

├── package.json            // The package.json file containing all the dependencies and scripts
├── package-lock.json       // Locks the version of the packages installed through npm
├── public                  // This directory contains all the public files
└── src                     // The main directory that contains all the app specific code           
    ├── assets              // This directory contains all the assets necessary for the app
    │   ├── css             
    │   └── images
    └── components          // This directory contains all the components used in the App
        └── designs         // All the available designs are available in this component

Setup the app

  • Install npm.
  • Install all the required javascript dependencies 
    npm install
  • Build for Production 
    npm run build
After building for production it will create a build directory in the root directory. You will upload the content of this directory to your server.
P.S. Everytime you change anything in the source code you must run 
build
and upload the updated content to your server.

Grid system

"Formalin" is based upon Twitter Bootstrap css framework that is very easy to use and adapt. It follows the mobile first approach to make the website readily available for all devices and screen sizes. Bootstrap grid is based upon 12 columns that are well defined in the bootstrap.css file inside the css folder. You can get to know about the grid system in detail from this url https://getbootstrap.com/docs/4.0/layout/grid/

We used a package called reactstrap to implement the Bootstrap 4 Grid into the template. You can find out more about the package here: http://reactstrap.github.io/.

Icons font

Font Awesome

"Formalin" uses font awesome icon fonts for its icons requirement and no images has been used for any icon. A complete list of icons along with the class names can be found here: https://fontawesome.com/icons

We used the react-fontawesome package to implement the FontAwesome Icons into the template. To know how to use the package goto this link.


Designs

"Formalin" provides 10 designs that you can choose from. You can find all the designs in the src/components/designs directory. In the demo we are using a state to switch between designs but in your use case you might want to just render the design component that you need. The available options are:

  • Default
  • Edge
  • Median
  • Night
  • Owl
  • Comet
  • Half
  • Incomplete
  • Snowy
